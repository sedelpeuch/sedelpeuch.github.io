Exercice 1​

  1. Une première version naïve serait d'utiliser un algortihme de type brute force. On part des nnn sommets et à chaque itération on appelle l'algorithme sur les n−1n-1n−1 arêtes possibles etc. Cela nous donne n!n!n! possibilités, soit la pire complexité possible. Pour résoudre cet algorithme il faut un moyen de générer les permutations, l'algorithme de heap itératif est utilisable.
// Pour la version itérative, on simule le processus récursif en mémorisant les indices de boucle dans un tableau compteur.

Heap_iteratif(n : entier, A : tableau) :

    // i indique le niveau de la boucle en cours d'incrémentation
    i := 0;

            Si i est pair :
                échanger A[0] et A[i]
            Sinon :
                échanger A[compteur[i]], A[i]
            Fin Si

Exercice 2​

Nous pouvons utiliser le cours et le théorème de Dirac pour le montrer.

Théorème de Dirac : Un graphe à nnn sommets (avec n>3n>3n>3) dont chaque sommet est au moins de degré n2\dfrac{n}{2}2n​ est hamiltonien.

Pour rappel un graphe complet est un graphe simple dont tous les sommets sont adjacents deux à deux. Chaque sommet à donc un degrés de n−1>n/2n-1 > n/2n−1>n/2 pour n>3n>3n>3. Ainsi tout graphe complet est hamiltonien.

Comme dans l'exercice 1 nous avons discriminé deux méthodes pour déterminer si un cycle est hamiltonien nous pouvons aussi discriminer deux méthodes exactes pour résoudre le problème du voyageur de commerce. Dans un premier temps nous pouvons utilser un pseudo code naïf. Cet algorithme à une complexité de O(n!), plus précisément (n-1)!/2 si on pense à ne pas recalculer ce que l'on a déjà calculé. On admet qu'il existe une fonction permutation permettant de fournir une permuttation d'un vecteur si il en existe une, zéro si elle ont toutes déjà été donnée.

Exercice 4​

On se place dans le cas où les coûts sont des déplacements et donc c(u,v)=c(u,w)+c(w,v)c(u,v)=c(u,w)+c(w,v)c(u,v)=c(u,w)+c(w,v)

Entrée : G=(V,E) un graphe complet et c(i,j) une fonction donnant le coup entre les villes i et j
Sortie : un chemin de coût minimal
Choisir un sommet r dans V comme ville initiale
Construire un arbre couvrant minimal T dans G à partir de r via l'algorithme de Krustal
Retourner le cycle hamiltonien H qui visite les sommets dans l'ordre trouvé par l'algorithme de Krustal

On veut maintenant prouver que l'algorithme est un algorithme d'approximation de facteur 2. On note C∗C^\astC∗ la tournée optimale et CCC la tournée courante. On cherche donc à montrer que c(C)≤2c(C∗)c(C)\leq2 c(C^\ast)c(C)≤2c(C∗). Soit TTT l'arbre couvrant minimal, on a c(T)≤c(C∗)c(T)\leq c(C^\ast)c(T)≤c(C∗) (puisque C est construit avec l'algorithme de Krustal, C privé de l'arète qui forme le cycle est un arbre). Si l'on note PPP la liste formé par le parcours de TTT on obtient c(P)=2c(T)c(P)=2c(T)c(P)=2c(T) car chaque arête est visitée deux fois. On remarque que l'on peut supprimer un sommet de PPP (facilement mis en évidence avec un arbre à 3 sommets). Ainsi c(P)≤2c(C∗)c(P)\leq 2c(C^\ast)c(P)≤2c(C∗) et donc c(C)≤c(P)≤2c(C∗)c(C)\leq c(P) \leq 2 c(C^\ast)c(C)≤c(P)≤2c(C∗). L'algorithme est donc un algorithme d'approximation de facteur 2.
